================================================================================
ARCHITECTURAL DECISIONS
================================================================================

Why Not Python FastAPI Bridge?
  → Direct Java ONNX integration is simpler and faster
  → Eliminates network latency between Java and Python
  → Single deployment artifact (one Java process)
  → Easier operations and monitoring

Why Not Pickle Format?
  → ONNX is framework-agnostic
  → Better for Java integration
  → Standardized model format
  → Future-proof for other languages

Feature Extraction in Java (Not Python Bridge)?
  → Ensures determinism in all scenarios
  → Matches Python exactly (fully tested)
  → No network round-trips
  → Easier debugging

Singleton OnnxModelService?
  → Model is expensive to load
  → Thread-safe environment and session
  → Loaded once at application startup
  → Reused for all inference requests

Synchronous Inference?
  → Level-1 detection must be fast (< 200ms)
  → No need for async complexity
  → Simpler exception handling
  → Easier debugging and monitoring
